RoboJournal ChangeLog

Version 0.3

Release Date: 8/28/2012

The following changes were made in this release:

  1. Changed RoboJournal logo image in FirstRun and AboutRJ classes.
  2. Fixed bugs in JournalCreator class:
    1. Bugfix: System username is now displayed on Linux/Unix systems.
    2. Bugfix: Added more padding below the "Passwords [do not] match" text. This keeps the text from being too close to the bottom of the frame.
    3. Added tooltips to the Journal Name, Journal Host, and Username fields.
  3. Simplified and changed text on FirstRun class UI file. This solves a bug where the intro paragraph text size changed depending on system font size.
  4. Added Build Helper Script (for Windows only).
  5. Bugfix: Fixed a bug that prevented the current color in use from being autoselected in the Config class color chooser dialog.
  6. Bugfix: Fixed incorrect error reporting when journal creation process fails due to a missing Qt MySQL driver.
  7. Bugfix: Added an error message that is triggered if you try to create a journal where journal@host already exists.
  8. Bugfix: Changed project file to ensure that builds are compiled in release mode by default on Windows systems.
  9. Added a shortcut/menu icon (F11) to expand/collapse entry tree.
  10. Added a check for Caps Lock to DBLogin class.
  11. Changed exit warnings in Editor class.
  12. Bugfix: Removed extra padding on MainWindow::Toolbar when buttons are set to image-only.
  13. Modified DBLogin Class:
    1. Removed root login warning label and replaced it with QMessageBox warnings/errors. This helps reduce crowding/sizing problems on large-text systems.
    2. TextBoxes are set to QLineEdit::ReadOnly instead of QLineEdit::SetDisabled when UseDefaults==true.
  14. Changed text for error (critical) QMessageBoxes in Editor class.
  15. Fixed program icon for Linux systems (previous versions used the too-small Windows icon that looked bad when stretched).
  16. Added journal export system to export database entries (or the whole database) to HTML or plain text:
    1. Added Save button to toolbar.
    2. Added Save entry and CRTL+S shortcut to File menu.
  17. Removed expand/collapse widget on root node (journal name) in entry tree.
  18. Changed MainWindow::Statusbar text:
    1. Reworded the welcome message (Click Connect (or press F2) to begin working with a journal database.) and the New Journal message (Welcome to your new journal! Click Write in Journal (or press F4) to get started.).
    2. Separated Username text into its own pane.
    3. Added images to Username and Entry Count panes.
  19. Config class changes:
    1. Enlarged Config window to reduce crowding in some GNOME setups.
    2. Added Apply button to Config::QDialogButtonBox.
    3. Made auto-load feature a config menu option. This lets the user decide if he/she wants the most recent entry to load automatically after login.
  20. Added tagging system (Tagger class):
    1. Modified MySQLCore::Update() to prevent NULL data from being written to the tags column in database entries table.
  21. Added JournalSelector class:
    1. FirstRun class now calls JournalSelector if user decides to use existing journal during config.
    2. Added a private function that executes a SHOW DATABASES tranaction to MySQLCore class.
  22. Changed window icon to database_add.png in JournalCreator class.
  23. Rewrote changelog.html from scratch to be XHTML 1.0 compliant.
  24. Replaced original compile/installation documentation (INSTALL-GUIDE-r3.htm) with an XHTML 1.0 compliant document (compile-instructions.xhtml).

Version 0.2.1

Release Date: 5/11/2012

The following changes were made in this release:

  1. Added "make install" and "make uninstall" support for Linux/OS X (RoboJournal gets installed to /usr/local/bin).
  2. Added Linux/OS X installation instructions to INSTALL-GUIDE-r3.htm.
  3. Reformatted Changelog.htm to pass HTML 4.01/STRICT.
  4. Removed MySQL header dependencies in source to ensure full GPL3 compliance.
  5. Removed unnecessary source and header references to QtSql/QMYSQLDriver and QtSql/QPSQLDriver. At this moment, QtSql/QSqlDatabase is all that is needed.
  6. Removed unnecessary reference to CONFIG += qsqlmysql from robojournal.pro.
  7. Removed unused/legacy code (headers, cpp, and ui) from source package.
